### What shipped

Replicates the [Conexsol Florida Services Trello board](https://trello.com/b/eBhmHKjv) (now the standard for TX and GA) as a new Kanban view in Service Orders.

- **`Job.pipelineStage?`**, a new OPTIONAL field carrying an 11-stage funnel: leads, needs_first_quote, first_quote_in_progress, site_transfer_processing, site_transfer_completed, service_quote_in_progress, needs_scheduling, done, email_follow_up, closed_won, closed_archived.
- Deliberately **orthogonal to `status`/`woStatus`**. The funnel tracks the sales and site-transfer pipeline; status tracks execution. Nothing in billing, `WO_TO_JOB_STATUS`, or `CONTRACTOR_VISIBLE_STATUSES` reads `pipelineStage`, so the existing production behavior cannot regress.
- **S1 tab**, leftmost in the view toggle (S1, Kanban, List, Calendar, Map). 12 columns: Unstaged + the 11 stages.
- Completed column auto-populates from `pipelineStage === 'done' || status === 'completed'`, so finished orders surface without a drag. Unstaged excludes `status === 'completed'`.
- Lead Lobby site-transfer conversion stamps `site_transfer_processing`.

## INCIDENT: types/index.ts silently truncated, broke the prod build

`32b7bec` rewrote `types/index.ts` from **998 lines down to 859**, dropping 10 exports that were never meant to be touched: `WorkOrder`, `WorkOrderStatus`, `WorkOrderType`, `WorkOrderLineItem`, `ClientProfitability`, `OperationsStats`, `CRMState`, `LeaderboardEntry`, `PriceList`, `UserRoleCRM`. The commit's own diffstat said "40 insertions, 179 deletions" for a change that should have been additive only.

Vercel failed with `TS2305: Module '"../types"' has no exported member 'WorkOrder'` (5 errors, all in `lib/operationsStore.ts`).

**Why local `npm run build` passed and Vercel failed:** the working tree had *uncommitted* edits to `operationsStore.ts` that no longer import those types. Vercel builds the **committed** tree, which still does. The dirty working tree masked the breakage completely.

**Fix:** restored `types/index.ts` from `c40f4f1` and re-applied only the two pipeline additions. Diff vs `c40f4f1` is now 39 insertions, 0 deletions.

Production was never serving the broken build: the Vercel build errored, so the previous good deployment stayed aliased. Only `main`'s HEAD was broken, for roughly one hour.

### Patterns worth reusing

- **A green local build proves nothing when the working tree is dirty.** To replicate what Vercel actually compiles, either `git stash push -- <dirty files>` and build in place, or `git worktree add --detach <path> HEAD`. Restore the stash immediately after. Verified this run: clean tree gave `tsc -b` + `vite build` clean and 211/211 vitest pass.
- **Check the diffstat sign before committing.** An additive change reporting large deletions is the signal. `git diff <base> -- <file> | grep '^-' | grep -v '^---' | wc -l` should be 0 for a pure addition.
- **Version tags do not change what the app reports.** `vite.config.ts` stamps `package.json`'s `version` into `public/version.json`. Tagging `v1.7.5.5` while `package.json` still said `1.7.5.4` left the live app reporting the old version. Bump `package.json`, not just the tag.
- The production alias is **`solarflow-dashboard-sooty.vercel.app`**, not `solarops.vercel.app` (that 404s). Find it with `vercel inspect <deployment-url>` and read the Aliases block.
