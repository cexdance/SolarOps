## API fault-tolerance system + serverless crash fix (DEPLOYED b82f024)

### Commits pushed to main
- `b82f024` fix(api): prevent serverless crash on missing env + add fault-tolerance layers
- `ff7ae75` feat(admin): User Permissions panel in Settings (pre-existing branch work, shipped same push)
- Range: `731f4bd..b82f024` on `main`.

### Root cause (the outage)
Production returned `FUNCTION_INVOCATION_FAILED` on `/api/solaredge`, `/api/trello-card`, and every other guarded proxy at once. A missing `SUPABASE_SERVICE_ROLE_KEY` made `api/_auth.ts` call `createClient('')` at module import time, which throws (`supabaseKey is required.`). Because the throw was at import, all 7 endpoints importing the shared auth guard crashed before any handler code ran. The graceful `if (!SERVICE_ROLE_KEY)` 500 guard was dead code below the crash point.

### Fix: defense-in-depth (5 layers)
1. `api/_auth.ts` - lazy-init the Supabase admin client behind the key guard, so a missing key returns a clean `500 {"error":"Server auth not configured"}` instead of crashing on import.
2. `api/_env.ts` - central env contract (single source of truth, presence only, never secret values). `env()` treats empty/whitespace as absent.
3. `api/health.ts` - `GET /api/health` returns `200 ok` / `200 degraded` / `503 critical` + live Supabase auth probe. Monitoring warning light.
4. `scripts/check-env.mjs` - build guard. Blocks prod deploy when a critical var is missing (strict under `VERCEL_ENV=production` or `STRICT_ENV=1`; lenient locally). Wired into `build` + `build:prod`.
5. `src/lib/resilientFetch.ts` - client retry+backoff with failure classification (transient/config/auth/quota/client). Wired into `SolarEdgeMonitoring.fetchAlertCounts` so a transient/config failure keeps cached alert counts instead of blanking the view.

Regression test: `src/__tests__/apiResilience.test.ts` reproduces the import-time crash and pins graceful degradation for all guarded handlers (12 tests). Full suite 121/121, `tsc -b` clean.

### Files modified
- Modified: `api/_auth.ts`, `package.json`, `src/components/SolarEdgeMonitoring.tsx`
- Added: `api/_env.ts`, `api/health.ts`, `api/RESILIENCE.md`, `scripts/check-env.mjs`, `src/lib/resilientFetch.ts`, `src/__tests__/apiResilience.test.ts`
- (ff7ae75 also added `src/components/admin/UserPermissionsPanel.tsx`, `src/components/admin/UserActivityLog.tsx`)

### Patterns worth reusing
- Never construct external clients (`createClient`, etc.) at module top level in serverless handlers. Lazy-init behind a config guard so a missing env var degrades to a clean error instead of `FUNCTION_INVOCATION_FAILED`.
- Critical vs optional env split: critical = blocks deploy + 503 on health; optional = single feature degrades, rest of app fine.
- Build guard auto-arms on Vercel via `VERCEL_ENV=production`; no vercel.json change needed.
