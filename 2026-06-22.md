## Service Order card + panel: age priority, card cleanup, comments-first overview

**Commit:** 6acc277 (pushed to main)

### What changed and why
1. **JobCard age-based priority counter** (`Jobs.tsx` `derivedPriority`): priority is now derived, not stored. Defaults to **Low** and bumps one level every **3 days** (Low -> Medium -> High -> Critical, capped). Open orders count off `createdAt`; invoiced orders reset the counter off `invoicedAt` (chase payment); completed/paid/archived stay Low. Replaces the old weeks-based logic. Helper `escalate(since, now)` = `floor(elapsed / 3 days)` clamped to the 4-level array.
2. **JobCard cleanup**: removed the `{job.serviceType}` chip (the long "Site visit - Simple / inspection / quick review" gray box = the "second explanation"). Only the `woCategory` chip remains (Site Visit / Optimizer / Inverter). Dropped now-unused `serviceTypes` array + `ServiceType` import.
3. **JobCard header reorder**: US-15XXX client badge moved to its own row **above** the customer name (was inline right of name). Falls back to `customer.clientId` when `job.clientId` is absent.
4. **ServiceOrderPanel comments-first**: moved the Comments & Activity block to the **top of the Overview tab, before Priority** so staff see WO activity at a glance. Same `ActivityFeed` + `MentionTextarea` thread format (reactions, @mentions, edit/delete, image paste). Swapped leading `border-t` for `border-b`. The Comments *tab* was already removed in prior uncommitted work; the feed now lives only in Overview.
