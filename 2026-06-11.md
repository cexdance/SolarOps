## DEPLOYED c57b018 - CRITICAL: contractors can never reach the staff app + View-as React #426 fix

User reported a **critical security breach**: contractor Jaime Mendez saw the ADMIN Dashboard on his phone. Plus React #426 when admin clicked "View as".

### Breach root cause
The app assumed ANY valid Supabase session = staff. Contractor mode lived only in fragile `sessionStorage` (`solarflow_contractor_mode`), which mobile Safari drops on tab reopen -> on refresh, a contractor's durable Supabase session fell through to the staff setup (`role: meta.role ?? 'admin'`, `setIsAuthenticated(true)`, no `isContractorMode`) = admin view. `finishStaffLogin` (staff login screen) also built a staff user with NO role check.

### Fix (role = source of truth, enforced at 3 entry points)
- NEW `src/lib/authRouting.ts`: pure `isContractorAccount(meta)` + `resolveSessionRoute(meta,email,contractors)` -> `{route:'staff'|'contractor'|'deny'}`. role==='contractor' -> approved match=contractor, else deny; NEVER staff. Dual-role staff (staff role + isContractor) stay staff.
- App.tsx (1) **session-restore effect** (the durable-refresh fix): contractor session routes to portal or signs out. (2) **finishStaffLogin**: rejects contractor on staff screen ("use the Contractor login"). (3) **render guard** before `renderView()`: blocks staff UI if a contractor-role user ever leaks through (defense in depth).
- NEW `src/__tests__/authRouting.test.ts`: 8 cases pinning "contractor-role session is NEVER staff". 146 tests green.
- COMMITTED TO MEMORY: `security_contractor_access_control.md` (CRITICAL, do-not-regress) + MEMORY.md top-of-index callout.

### View-as React #426 fix
The contractor portal (lazy `ContractorDashboard`) was returned BEFORE the page's Suspense boundary and swapped in by a discrete click -> "synchronous input suspended" (#426). Fixed: wrapped `handleViewAsContractor` state sets in `startTransition` + wrapped the contractor-mode return in its own `<Suspense>`. Verified in preview: View-as shows the banner + portal, no error overlay.
